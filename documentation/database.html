Note: In each relationship there is a primary table and a secondary table. If the table being described is the primary table, in a relationship, it will be a "has" relationship and "Enforce RI?" and "Cascade Delete?" will be indicated. Otherwise it will be a "looks up" or "belongs to" relationship and "Enforce RI?" and "Cascade Delete?" will not be indicated. For example, "one Agent has many Projects" indicates that tblAgent is the primary table in the relationship so "Enforce RI?" and "Cascade Delete" will be indicated. The corresponding relationship, "one Project looks up one Agent", will leave the columns for "Enforce RI?" and "Cascade Delete" blank.

Stored Queries

qryMaxUpdate
qryMaxUpdate returns a list of projectID's and the date of the most recent status update for each projectID. This query is used by qryMostRecentStatus.

SELECT tblStatus.intProjectID,
Max(tblStatus.datUpdate) AS MaxOfdatUpdate
FROM tblStatus
GROUP BY tblStatus.intProjectID;


qryMostRecentStatus
qryMostRecentStatus uses qryMaxUpdate to return a list of projectID's and the date and status of the most recent status update for each projectID. This query is used in generalReport.cfm, SCHReport.cfm and projectSummaryReport.cfm.

SELECT qryMaxUpdate.intProjectID,
Last(tblStatus.memStatus) AS LastOfmemStatus,
Last(tblStatus.datUpdate) AS LastOfdatUpdate
FROM tblStatus INNER JOIN qryMaxUpdate
ON (tblStatus.intProjectID = qryMaxUpdate.intProjectID)
AND (tblStatus.datUpdate = qryMaxUpdate.MaxOfdatUpdate)
GROUP BY qryMaxUpdate.intProjectID;
