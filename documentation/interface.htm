Overview

ROMPS is a project management system for REAS acquisition, disposal and consulting projects. The primary users are REAS advisors who input and maintain the data for their own projects. The system is also used by management to gain an overview of current and completed projects as well as finding information about specific projects.

In order to ensure that the system is useful for the advisors, ROMPS has been designed to be simple and intuitive. It was decided that a web application would provide the most familiar interface and would be the most widely accessible. It was also important to ensure that using the system would not be a burden to the advisors, so the amount of data maintained about each project has been kept to a minimum (see below).

To give management an overview of the current and completed projects, the system includes a basic reporting function. The user can search for projects using several different criteria and produce summary reports based on the results.

There is also a separate administration section where users with the appropriate access level can maintain the data that appears in many of the drop-down lists. Admin users can also download data from the database tables in ASCII format.

  • Login/Logout

    In order for a user to add or edit any data they must be logged in. Only users with the "read/write" or "admin" access levels can log in. To log in, the user simply clicks on "Menu" and then "Login". The "Login" option is only available when the user is on the Project Search or Search Results pages. Once at the Login page the user simply has to select their name from a list and enter the password and they will be returned to the Project Search page.

    To log out, the use clicks on "Menu" and then "Logout". Again, the "Logout" option is only available when the user is on the Project Search or Search Results pages. When the user clicks on "Logout" they will be returned to the Project Search page.

  • Search and Sort

    The Project Search page is the first page the user sees when they access the system. It allows the user to filter projects by project status, type, location, description, advisor, client, project number, file number, and client contact. When they have selected the their criteria from the drop-down lists and/or entered their criteria in the text boxes, they click "Submit" to display the results on the Search Results page, or "Reset" to reset all the search criteria. If the user is logged in, their name will automatically be selected in the "Advisor" drop-down list. Also, only items which actually exist as data for at least one project will be available in the list-boxes (i.e. if there are no projects with Vancouver as the location, Vancouver will not be displayed in the "Location" drop-down list).

    To sort the results the user can click on the column headings on the Search Results page. If they want to sort by two fields they can click on a second column which becomes the primary sort, and the first column becomes the secondary sort.

  • Project Display and Edit

    Projects fall into one of three categories: acquisitions, disposals and consulting projects. When the user wants to update a project, they perform a simple search (see above) and click on the "Details" button for the project of their choice. Depending on the type of the selected project, this will take them to the appropriate project page (either acquisitions.cfm, disposals.cfm, or consulting.cfm) in which they can flip through the tabs and add to, or edit, the project's data. When they are done they can either click on "Save" or "Cancel" to return to the search results and select another project. If the user is not logged in, only the "Cancel" button will be available and they will not be able to save any changes.

    There are two types of data and, thus, two types of data entry. There is some data that a project only has one of. For example, a project only has one description and one location. This type of data is simply entered into text boxes or selected from drop-down lists. The other type of data is data that a project can have many of. For example, a project can have many status updates and many appraisals. This second type of data requires the user to click on an add or edit button which will take them to another page. On this secondary page, they will enter the data into text boxes and drop-down lists and then click submit to add the new data or edit already existing data.

  • Print

    There are two types of information that can be printed: information concerning a particular project, and information concerning many projects. To print information concerning a particular project, the user must go to that project by performing a search (see above) and clicking on the "Details" button for that project (see above). Once at the project, the user clicks on "Print" to see a list of the available reports that can be printed. This list will vary, depending on the project type, as follows:

      Acquisition Status Journal
        Acquisition Summary
         
      Disposal Status Journal
        Disposal Summary
        Fact Sheet
        First Nations Consultations
         
      Consulting Status Journal

    To print information about many projects, the user must create the list of projects they want to include in the report by using the Project Search and selecting the appropriate criteria. Then the user must click on "Print" and click on the report of their choice. The three reports that are available are "SCH Report", "General Report" and "Project Summary". By clicking on "SCH Report" or "General Report", the user is taken to another page (SCHReportCreate.cfm or generalReportCreate.cfm) where they can choose the sort and enter other data that will appear on the report. The user must then click on "Submit" to create the report which can be printed from the browser. If the user chooses the "Project Summary" report (projectSummaryReport.cfm), the report is immediately created and can be printed from the browser. For this report the projects are sorted in the same order as the Search Results.

  • Admin

    The Admin section is where users can add, remove and edit items in the drop-down lists that appear in the project pages. To access the Admin section, users must be logged in and then click on "Admin" at the top of every page. Users with both read/write and administrative access can edit the "Location", "Client/Department" and "Client Contact" lists, as well as change their passwords. In addition to this, users with administrative access can also edit the "Advisors", "Property Types", "Interest Sold Types", "Disposal Methods", and "Purchaser Types" lists. Admin users can also delete projects from the system.

acquisitions.cfm

acquisitions.cfm displays the project information for acquisition projects. When the user clicks on the "Details" button of an acquisition project from the Search Results, acquisition.cfm is called. It reads the Project ID from the querystring to determine which project should be displayed and then retrieves the data from tblAcquisitions and tblProjects and fills in the fields with this data. acquisitions.cfm can be requested from many other files and depending on the parameters passed to it, it may perform various update and append queries. When acquisitions.cfm is requested by projectList.cfm or newProject.cfm it saves all of it's data in Session variables so that if the user clicks "Cancel" projectList.cfm can restore all the data to it's original state.

addOptions.cfm

When a user is adding a new project they have the option of adding new locations, client/departments and client contacts. When they click on a button to add one of these options, the form on newProject.cfm is sent to addOptions.cfm, where the values are entered into hidden form fields. login.cfm

When the user fills out the form and clicks submit, the form is sent to projectSearch.cfm along with the hidden field, Form.LoggingOn. This tells projectSearch.cfm to attempt to log in the user. If the user information is correct, the appropriate client variables are set to log in the user, otherwise projectSearch.cfm requests login.cfm?failed=yes and the process is repeated.

Files that can request login.cfm:

Every file that has header.cfm included in it can request login.cfm as long as they are not logged in already.


Files that login.cfm can request:

File QueryString Parameters and Form Fields
projectSearch.cfm [Form.LoggingOn | URL.Logout]

newProject.cfm

newProject.cfm is where the user creates a new project. Except for the Project Type drop-down list, all the fields on this page are common to all three project types. The user may add new options to three of the drop-down lists: Location, Client/Department, and Client Contact. When the user clicks on one of the "Add New" buttons they will be taken to addOptions.cfm. When they return, the option they added will automatically be selected. When they have finished filling in the data for their new project, the user can click on "Submit" or "Cancel". If they choose submit the form is sent to newProjectQuery.cfm which creates the new project and sends the user to the appropriate page (acquisitions.cfm, disposals.cfm, or consulting.cfm).

newProjectQuery.cfm

newProjectQuery.cfm is called when a user clicks on "Submit" on newProject.cfm. It reads Form.projectType and performs the appropriate queries to create a new project of the type specified in this field. The data from the other fields from newProject.cfm is also entered into the project.

projectList.cfm

projectList.cfm displays all the projects that meet the criteria entered in projectSearch.cfm. The form data from projectSearch.cfm is saved into client variables so that if projectList.cfm is requested by a page other than projectSearch.cfm, it can still generate the same search query. projectList.cfm generates a link for each project consisting of the appropriate project page (either acquisitions.cfm, disposals.cfm or consulting.cfm) followed by the "projectID" querystring parameter. The user can sort the projects by clicking on the column headings. When this happens, projectList.cfm sends a form to itself with Form.sort containing the field to be sorted by. If a user clicks on "Cancel" on one of the project pages, projectList.cfm performs queries to restore the projects data to that which is saved in the session variables.

projectSearch.cfm

projectSearch.cfm is the first page the user should see. It has two functions. The first is to let the user search for projects. The second function is to attempt log in the user if they have submited their login info on login.cfm, and to log out the user when they click "Logout". If an attempted log in fails, projectSearch.cfm will redirect the user to login.cfm?failed=yes.

statusPrint.cfm

statusPrint.cfm displays the Status Updates for a specific project in a printable format. If the user is logged in, then clicking on "Status Journal " on the "Print" menu from the project pages (acquisitions.cfm, disposals.cfm, and consulting.cfm) will request the project's save page (acquisitionSave.cfm, disposalSave.cfm, or consultingSave.cfm) which, in turn, will request statusPrint.cfm. If the user is not logged in, they will be taken directly to statusPrint.cfm to avoid saving changes they may have made to the project.

Files that can request statusPrint.cfm:

File QueryString Parameters and Form Fields
acquisitions.cfm URL.projectID
acquisitionSave.cfm URL.projectID
disposals.cfm URL.projectID
disposalSave.cfm URL.projectID
consulting.cfm URL.projectID
consultingSave.cfm URL.projectID


Files that statusPrint.cfm can request:

statusPrint.cfm cannot request any pages, users must use the back button on their browser to return to the project page. Note that, because acquisitionSave.cfm, disposalSave.cfm and consultingSave.cfm do not send any information to the browser, the back button does not return the user to these pages.

statusUpdate.cfm

Projects of all types can have many status updates. When the user clicks on the "Add New" or "Edit" buttons on status section of the project pages, they are taken to statusUpdate.cfm (via acquisitionSave.cfm, disposalSave.cfm, or consultingSave.cfm) where they fill out a simple form and click either "Submit" or "Cancel". The form is sent to the page specified in URL.type (either acquisitions.cfm, disposals.cfm.cfm or consulting.cfm).

disposalSave.cfm URL.type(disposals), URL.projectID, URL.add | URL.statusID
consultingSave.cfm URL.type(consulting), URL.projectID, URL.add | URL.statusID
